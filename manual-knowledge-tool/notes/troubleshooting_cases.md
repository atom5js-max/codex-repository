## CASE-001 | 인버터 S300 속도지령 불응 (2024-03)

**장비**: LS S300 인버터 / 현장: 권취기 라인 2호기  
**증상**: PLC Modbus 통신으로 주파수지령을 Write해도 인버터가 속도를 바꾸지 않음.  
**원인**: 주파수지령 소스 파라미터(DRV-05)가 Keypad로 설정되어 있었음. 통신으로 바꾸려면 Frequency Reference = Communication(3)으로 설정해야 함.  
**조치**: DRV-05 → 3 (Communication) 변경, DRV-06(운전지령소스)도 RS-485로 변경  
**참고**: 운전지령(Run Command)과 주파수지령(Frequency Reference) 소스가 별도 파라미터임을 주의.

## CASE-002 | 로드셀 증폭기 출력 흔들림 (2024-04)

**장비**: HBM C2 로드셀 + 증폭기 / 현장: 장력측정 스탠드  
**증상**: 무부하 상태에서 PLC AI count가 ±15 count 변동. 정상 범위는 ±2 이내여야 함.  
**원인**: 쉴드 케이블이 양쪽 모두 접지되어 Ground Loop 형성됨.  
**조치**: 증폭기 측 쉴드 접지선 제거, 현장측(로드셀 쪽)만 단일 접지로 변경.  
**결과**: 변동폭 ±1 count로 개선.  
**비고**: 기계 진동이 심한 경우 증폭기의 Low-Pass Filter 상수를 늘려야 할 수 있음.

## CASE-003 | Modbus RTU 통신 불안정 (2024-05)

**장비**: LS XGB PLC ↔ 인버터 3대 / 현장: 압연기 라인  
**증상**: 인버터 1번은 통신 정상, 2번은 간헐적 에러, 3번은 항상 에러.  
**진단**:
- 국번 확인: 1번=1, 2번=1 (중복!) → 2번 국번을 2로 변경
- 3번은 종단저항 미설치 → 120Ω 종단저항 추가
**조치**: 국번 중복 해소 + 종단저항 설치 후 정상화  
**교훈**: RS-485 버스 끝단 장치에는 반드시 종단저항 설치. 국번 중복 시 응답 충돌로 CRC 에러 발생.

## CASE-004 | LabVIEW DB Insert 지연 (2024-06)

**장비**: LabVIEW 2022 + SQL Server / 현장: 시험설비 데이터로거  
**증상**: 100ms 주기 수집인데 1~2초마다 데이터가 한꺼번에 Insert됨. 간헐적 데이터 누락.  
**원인**:
1. DB 연결을 Insert 루프마다 Open/Close하는 구조 → 연결 오버헤드 누적
2. Execute SQL 방식 사용 (Prepared Statement 미사용) → 매번 쿼리 파싱
**조치**:
1. DB 연결을 프로그램 초기화 시 1회만 Open, 종료 시 Close
2. Prepare Statement를 루프 밖으로 이동
3. 100건 단위로 Transaction 묶어서 Commit
**결과**: 평균 Insert 지연 80ms → 12ms로 감소, 데이터 누락 없음.

## CASE-005 | 전력량계 펄스 계산 오차 (2024-07)

**장비**: 오므론 전력량계 (1000 pulse/kWh) / 현장: 에너지 모니터링  
**증상**: PLC에서 계산한 적산 전력이 전력량계 표시값과 약 10% 차이 발생.  
**원인**: PLC 프로그램에서 펄스 레이트를 100 pulse/kWh로 잘못 설정.  
**조치**: 전력량계 데이터시트 재확인 → 1000 pulse/kWh로 수정.  
계산식: kWh = 펄스 수 / 1000  
**비고**: 전력량계 사양서의 Pulse Rate(또는 Pulse Constant) 항목 반드시 확인.

## CASE-006 | PLC 아날로그 입력 스케일링 오차 (2024-08)

**장비**: Mitsubishi iQ-R PLC + 4-20mA 압력 센서  
**증상**: 압력 0 kPa일 때 PLC 계산값이 음수로 나옴.  
**원인**: AI 모듈 입력 범위가 0-20mA로 설정되어 있었음 (4-20mA가 아님).  
**조치**: AI 모듈 입력 범위를 4-20mA로 변경 후 오프셋 재조정.  
**스케일링 공식**:  
```
공학값 = (count - 0) × (최대값 - 최소값) / (16000 - 0) + 최소값
```  
※ 4-20mA 모듈에서 count 0 = 4mA, count 16000 = 20mA 인 경우 (모듈마다 다름)

## CASE-007 | 인버터 OV 트립 (과전압) (2024-09)

**장비**: LS S100 인버터 / 현장: 컨베이어  
**증상**: 감속 시 과전압(OV) 트립 발생.  
**원인**: 감속 시 전동기 회생에너지가 직류링크(DC Link)에 충전되어 과전압 트립.  
**조치 검토 항목**:
- 감속 시간(Decel Time) 증가 검토
- 제동저항(Braking Resistor) 설치 검토
- 동적제동 유닛(DBU) 용량 확인
**비고**: OV 트립 레벨은 보통 DC 800V 이상. 실제 DC 링크 전압 모니터링 권장.

## CASE-008 | RS-485 신호 반전 (2024-10)

**장비**: 계측기 ↔ PLC RS-485 통신  
**증상**: 통신 케이블 연결 후 전혀 통신이 안 됨.  
**원인**: RS-485 A(+)/B(-) 케이블이 반대로 연결됨.  
**확인 방법**: 테스터기로 정상 통신 중 A단자는 B단자보다 전압이 높아야 함 (보통 +수백mV).  
**조치**: A/B 케이블 교체 후 즉시 통신 복구.
